> CCC 是一个多包仓库，按需取用。

CCC 以一组独立的 npm 包发布，只需安装项目实际用到的包即可。

<Callout type="info">
  大多数项目只需一个入口包。Node.js 环境使用 `@ckb-ccc/shell`，React 应用使用 `@ckb-ccc/connector-react`，需要自定义钱包界面时使用 `@ckb-ccc/ccc`。
</Callout>

## 功能速览

CCC 提供了多个包来满足不同开发场景的需求。根据你的开发环境（前端/后端）、框架选择（React/其他）和功能需求（钱包集成/协议支持），选择合适的包可以大幅提升开发效率。

- **核心包**：`@ckb-ccc/core` 和 `@ckb-ccc/shell` 提供基础功能，适合深度定制或后端开发。
- **钱包集成**：`@ckb-ccc/ccc` 聚合所有钱包，适合自定义 UI；`@ckb-ccc/connector-react` 提供预构建组件，适合 React 快速集成。
- **协议 SDK**：`@ckb-ccc/spore`、`@ckb-ccc/udt`、`@ckb-ccc/ssri` 支持特定的 CKB 协议标准。
- **单钱包集成**：针对特定钱包（JoyID、OKX、UniSat 等）的独立包，按需选择。

## 导入方式

所有包均在同一 `ccc` 命名空间对象上暴露公开 API：

```typescript
import { ccc } from "@ckb-ccc/<package-name>";
```

高级（不稳定）API 可通过 `cccA` 访问：

```typescript
import { cccA } from "@ckb-ccc/<package-name>/advanced";
```

<Callout type="info">
  `tsconfig.json` 中 `moduleResolution` 须设为 `node16`、`nodenext` 或 `bundler`，且不能禁用 `resolvePackageJsonExports`。CCC 使用 [Package Entry Points](https://nodejs.org/api/packages.html#packages_package_entry_points) 实现 Tree Shaking。
</Callout>
